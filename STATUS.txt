┌──────────────────────────────────────────────────────────────────────────────┐
│                        🎓 BEST PRACTICES                                     │
└──────────────────────────────────────────────────────────────────────────────┘

  For Stock Data:
  ✅ Use "Business Days Only" for weekend handling
  ✅ Enable outlier detection (Z-score, threshold: 3.0)
  ✅ Let auto-tuning select the best model
  ✅ Set forecast horizon to 4-12 weeks

  For Seasonal Patterns:
  ✅ Use SARIMA or Holt-Winters models
  ✅ Increase max_P and max_Q for seasonal capture
  ✅ Set seasonal period to match data (52 for weekly)

  For Manual Tuning:
  ✅ Start simple: ARIMA (1, 1, 1)
  ✅ Add seasonality: (1, 1, 1, 52)
  ✅ Increase p, q gradually if needed
  ✅ Use differencing (d) for stationarity

  For High-Volatility Stocks:
  ✅ Lower outlier threshold (2.0-2.5)
  ✅ Consider data smoothing
  ✅ Use wider confidence intervals
  ✅ Shorter forecast horizons (4-8 weeks)

┌──────────────────────────────────────────────────────────────────────────────┐
│                        🐛 TROUBLESHOOTING                                    │
└──────────────────────────────────────────────────────────────────────────────┘

  Problem: "Forecast generation timed out"
  ├── Reduce max_p, max_q in advanced options
  ├── Use manual parameters (skip auto-tuning)
  └── Try simpler model (ARIMA vs SARIMA)

  Problem: "No forecast displayed"
  ├── Check browser console (F12)
  ├── Check Docker logs: docker logs perbillion-ml-engine
  └── Verify health: curl http://localhost:3000/health

  Problem: "Data unsuitable for modeling"
  ├── Need at least 50 observations
  ├── Try different ticker with more history
  └── Adjust preprocessing options
